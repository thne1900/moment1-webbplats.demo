Arbetsprocessen:

Initialt

Här kommer en beskrivning på hur jag initialt började mitt projekt (webbplats demo) kring automaticering och publicering som Moment 1 bygger på.

  • Jag började med att skapade en katalog för projektet: "moment1-webbplats.demo" och öppnade upp denna i VSC (Visual Studio Code).
  • I terminalen skrev jag in ett kommando, för att öppna upp ett nytt "Node Package Manager"-projekt (npm init).
  • Efter detta kom "package.json"-filen upp i VSC, innehållande inställningar för projektet. Jag kontrollerade dessa uppgifter och ordnade upp dem, några togs bort och några lades till.
  • Därefter gjorde jag en installation med parcel, via terminalen (npm install parcel --save-dev). Packetkataloger/filer kom då upp och hamnade i fil-listan för projektet, som ex node_modules.
  • Jag skapade en ny fil (.gitignore) till de filer som inte ska versionshanteras. Placerade därför node_modules och package-lock.json filerna där då dessa skulle exkluderas från versionshantering.
  • I "package.json"-filen lades inställningar kring "source" in, från vilken dokument webbplatsen ska starta ifrån. Även kommandon som "start" och "build" lades till, för att kunna nå den kommande webbplatsen och även att kunna bygga ihop den och dess ihoppackade filer inför publicering.
  • Efter det, gjorde jag en katalog "src", då det är den parcel utgår ifrån, innehållande mina arbetsfiler. Så i denna src-katalog skapade jag de filer/mappar som webbplatsen ska komma att innehålla (html, css, js, images).
  • Sedan skrev jag in lite grundkoder i html och css-filen för att kunna testköra kommandon: "npm start" och "npm build", för att kontrollera att kommandona fungerade. Jag kunde nå den skapande URL:en vid körning av "npm start" och en katalog tillkom (dist) vid körning av kommandot "npm build", för den publiceringsklara webbplatsen.

Utvecklingen

Nedan följer en beskrivning på hur jag arbetade vidare på min webbplats och hur gången har sett ut.

  • Efter att projektets grund var skapad, med filer, kataloger och inställningar skapade jag ett nytt repo till GitHub (versionshantering) med samma namn.
  • Därefer gjorde jag ett inlogg på Netlify (publiceringsplattform), där mitt repo från GitHub skulle synkas till Netlify. Denna tillgång behövs för att kunna publicera min webbplats.
  • Vid hämtningen av repot från Github till Netlify, efter att grundinställningar bestämdes, dök ett felmeddelande upp som indikerade på fel vid "byggandet" av webbplatsen. Jag försökte då finna vad felet kunde vara, kontrollerade därför alla inställningar och koder med dess syntaxer. Det visade sig att "devDependencies":{"parcel": "^2.13.3"} saknades i package.json filen. Då körde jag installationen kring parcel på nytt och då såg det ut som tänkt och webbplatsen kunde "byggas".
  • När allt verkade fungera, börjades webbplatsens dokument utformas, med vad som skulle finnas med på sidorna med bla texter och bilder.
  • Jag testade även här att skapa en ny "branch"-Dev för att utforma mina dokument och testa innan de blev publicerade. Men misstänker att det blev någon miss när kopplingen till huvudgrenen skulle göras, då det uppkom ett felmeddelande. Därför valde jag att ta bort denna "dev".
  • Jag testkörde med några ändringar på bilder och tillägg i main-brachen för att se så att det blev rätt med synkningen till både Github och Netlify utan felmeddelanden.
  • När jag arbetade med bilderna, uppkom felmeddelande om att "Sharp" eventullt skulle användas, så därför testades detta att installera.
  • Jag valde här att skapa en ny "branch"-Dev1. I denna utformades dokumenten ytterligare med innehåll i respektive fil.
